# 篩選器 {#filters}

進一步了解Discover中可用的不同篩選器，以及如何使用這些篩選器。

>[!NOTE]
>
>Discover篩選器中的「符合使用者屬性」和「符合（進階）」運算子純粹是管理，可安全地忽略。

**帳戶Id**

_用於：帳戶型行銷_

從CRM選取或貼入一系列帳戶ID，以篩選結果。 帳戶ID比帳戶名稱提供更多唯一性，因為名稱可以是相同的。

**帳戶名稱**

_用於：帳戶型行銷_

從CRM中選取或貼入一系列帳戶名稱，以篩選結果。 字串可以有重複項目，因此可以有多個「[!DNL Marketo Measure]&quot;例如說明。 在此情況下，如果需要單一帳戶，請改用「帳戶ID」篩選。

**歸因模型**

_用於：概覽，行銷支出，廣告ROI，以帳戶為基礎的行銷，網路流量， CMO，付費媒體，內容行銷，護照_

選取要套用至展示板的單一歸因模型：首次接觸、銷售機會建立接觸、U形、W形、完整路徑或自訂模型。 並非所有層級都提供完整路徑和自訂模型。

**行銷活動**

_用於：概述，增長，廣告ROI，網路流量， CMO，付費媒體，內容行銷，護照_

依單一或多個促銷活動名稱篩選展示板。 運算子為篩選器提供額外的彈性，例如使用「包含」或「開頭為」運算子。 如果已套用管道或子管道篩選，則顯示的促銷活動清單將是套用篩選的子集。

**第1-10類**

_用於：概述，成長，廣告ROI, CMO，付費媒體，內容行銷， Velocity，快照，同類群組漏斗， Passport_

使用您在 [!DNL Marketo Measure] 設定。 您建立的類別清單會顯示在篩選器功能表中，因此，如果尚未設定類別，功能表中就不會有類別篩選器。 並非所有層級都提供區段類別，而可用的類別數量也因層級而異。

**管道**

_用於：概覽，增長，行銷支出，廣告ROI，網路流量， CMO，付費媒體，內容行銷， Velocity, Passport_

依單一或多個通道篩選展示板。 運算子為篩選器提供額外的彈性，例如使用「包含」或「開頭為」運算子。 輸入管道後，子管道和促銷活動篩選中顯示的值將來自套用的子管道篩選。

**同類群組階段**

_用於：同類群組漏斗_

選取您要檢視同類群組的階段。 您選取的階段會顯示在漏斗頂端，所有轉換都會從頂端向下流動。

**日期**

_用於：概覽，成長，行銷支出，廣告ROI，以帳戶為基礎的行銷，網路流量， CMO，付費媒體，內容行銷， Velocity，快照，同類群組漏斗， Passport_

選取日期範圍以篩選展示板中的資料，例如使用彈性的日期運算子，例如「在範圍內」、「在年內」或「在之前」。 例外是「快照」，您將在其中選擇一個日期以查看資料的快照。

**日期類型**

_用於：概覽，增長，行銷支出，廣告ROI，以帳戶為基礎的行銷，網路流量， CMO，付費媒體，內容行銷，護照_

選擇您要使用的日期類型，並系結至「日期」篩選器。 預設日期類型依主板而異。 Touchpoint Date指的是市場營銷活動發生的日期， Created Date是在CRM中建立Lead或Contact或Opportunity的日期， Close Date是Opportunity關閉的日期。

**維度**

_用於：付費媒體_

Dimension與「群組依據」功能類似，只是在付費媒體展示板上使用的方式稍有不同。 「Dimension」不會堆疊圖表，而會變更概述圖表中的線條，以及表格中的前導物件。

![](assets/1.png)

依預設，Dimension會設為子管道，並可變更為：

* 無：以匯總方式顯示所有內容，且無劃分
* 管道：依行銷管道列出資料
* 子頻道：依行銷子管道列出資料
* 促銷活動：依促銷活動列出資料
* 帳戶：按帳戶列出資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 廣告群組：依廣告群組列出資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 廣告：依廣告列出資料。 套用至Doubleclick廣告，因此若未使用Doubleclick，則不會顯示任何結果
* 廣告商：依廣告商列出資料。 套用至Doubleclick廣告商，因此若未使用Doubleclick，則不會顯示任何結果
* 創意：按創意列出資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 關鍵字：按關鍵字列出資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 版位：按位置列出資料。 套用至Doubleclick版位，因此若未使用Doubleclick，則不會顯示任何結果
* 網站：按站點列出資料。 套用至Doubleclick網站，因此若未使用Doubleclick，則不會顯示任何結果

**分組依據**

_用於：概覽，成長，行銷支出，以帳戶為基礎的行銷，網路流量， CMO_

調整圖表，以變更堆疊和分組在一起的維度。 依預設，「群組依據」會設為「管道」，可變更為：

* 無：以匯總方式顯示所有內容，且無劃分
* 管道：依行銷管道將資料分組
* 子頻道：依行銷子管道將資料分組
* 促銷活動：依促銷活動將資料分組
* 帳戶：依帳戶分組資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 廣告群組：依廣告群組分組資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 廣告：依廣告分組資料。 套用至Doubleclick廣告，因此若未使用Doubleclick，則不會顯示任何結果
* 廣告商：依廣告商分組資料。 套用至Doubleclick廣告商，因此若未使用Doubleclick，則不會顯示任何結果
* 創意：依創意將資料分組。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 關鍵字：依關鍵字分組資料。 套用至 [!DNL AdWords], [!DNL Bing]，和 [!DNL Facebook].
* 版位：依位置分組資料。 套用至Doubleclick版位，因此若未使用Doubleclick，則不會顯示任何結果
* 網站：依網站分組資料。 套用至Doubleclick網站，因此若未使用Doubleclick，則不會顯示任何結果

**登陸頁面**

_用於：內容行銷_

深入探討單一登錄頁面的效能，或可能包含特定字詞（例如「部落格」）的登錄頁面。

**量度**

_用於：概述，網路流量， CMO，付費媒體，內容行銷_

在不同的展示板上使用兩個不同的量度選擇器。 量度選擇器會變更圖表上的量度，以便您在檢視收入、支出或曝光之間切換。

在「概述」和CMO展示板上，提供與ROI量度相關的值簡短清單：

* 收入
* 支出
* 交易
* 管道收入
* 機會
* 聯繫人
* 銷售機會

在網路流量、付費媒體和內容行銷展示板上，有更多與ROI和漏斗量度相關的值清單：

* 收入
* 支出
* 交易
* 管道收入
* 機會
* 聯繫人
* 銷售機會
* 點按
* 曝光數
* 瀏覽
* 不重複造訪
* 頁面檢視
* Forms

**階段**

_用於：速度_

預設情況下，Velocity板顯示所有階段的時間，但要鑽取到特定階段，請使用「階段」篩選器選擇階段。

**子頻道**

_用於：概覽，成長，行銷支出，廣告ROI，網路流量， CMO，付費媒體，內容行銷，護照_

依單一或多個子通道篩選展示板。 運算子為篩選器提供額外的彈性，例如使用「包含」或「開頭為」運算子。 如果已套用管道篩選器，則顯示的子管道清單將是套用篩選器的子集。 輸入子管道後，「促銷活動」篩選器中顯示的值將來自套用的子管道篩選器。

**URL**

_用於：網路流量_

深入研究單一URL的流量，或可能包含「product」等特定字詞的URL。

**元**

_用於：速度_

預設情況下，Velocity板只報告已關閉的成功機會，但請調整此篩選器以查看已關閉成功或已關閉的失敗機會的速度。
