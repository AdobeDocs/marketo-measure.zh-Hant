# 接觸點欄位 {#touchpoint-fields}

過去，當客戶 [!DNL Marketo Measure] 如果我們沒有直接標籤整合，我們的客戶成功團隊會教導客戶如何適當地標籤其登陸頁面，讓他們運用正確的UTM格式，我們就能解析其廣告。 其中有些客戶不使用UTM，而是使用自己的標籤參數，這表示使用新的標籤結構，在其所有廣告網路中編輯其所有登錄頁面可能會非常耗時 [!DNL Marketo Measure] 執行。 為了適應其標籤結構，我們現在接受可與規則定義對應的自訂參數。 目標是要適應客戶對其自訂追蹤參數的使用，因此我們不需要要求他們變更其URL結構。

>[!AVAILABILITY]
>
>現已可在第2層和第3層使用完整細分功能。

>[!NOTE]
>
>這是進階功能，應僅由Professional Services設定。

## 啟用功能 {#enabling-the-feature}

從 [!DNL Marketo Measure] 設定功能表，導覽至接觸點欄位頁面。 從那裡，您可以選取 **是** 在 **啟用計算欄位**. 啟用後，您就可以自由建立接觸點欄位。

![](assets/one.png)

## 作法 {#how-to}

若要建立計算欄位，請記住，使用者可採取三種不同動作：擷取、對應至和串連。 這些也稱為定義計算欄位的運算子。

擷取

擷取運算子會從其他位置的欄位中提取值，例如：促銷活動欄位、銷售機會欄位，或在更進階的使用案例中， [從登錄頁面擷取自訂參數](https://docs.google.com/document/d/1NRViyCsXvPKbCTfGW32Yi2vWBjMDRF7bzkzKj9s2DDA/edit?ts=5e20b482#heading=h.xxwtissvw4){target="_blank"}. It then places it onto a Touchpoint Field (See [Maps To Example](https://docs.google.com/document/d/1NRViyCsXvPKbCTfGW32Yi2vWBjMDRF7bzkzKj9s2DDA/edit?ts=5e20b482#heading=h.xxwtissvw4){target="_blank"} #2)。

**範例#1**

Contact上有一個自訂欄位，即campaign_source__c，客戶想將其放至接觸點以用於報表用途。 您可以定義規則以建立名為「促銷活動來源」的計算欄位，並將值拖曳至該欄位。

目標：使用自訂欄位的值，並將其放入接觸點物件，以便更輕鬆製作報表。

* 建立計算欄位，並將其標示為「促銷活動來源」
* 從搜尋Contact.Campaign_Source__c欄位開始，定義規則
* 請使用運算子「擷取」，因為我們需要從參數中提取值
* 若要從欄位中擷取完整字串，我們將使用運算式「(.&#42;)&quot;

   * **(** 會標示擷取的開始
   * **)** 會標示擷取的結尾
   * **.&#42;** 告訴我們，我們正在提取

**範例#2**

此功能啟用的常見使用案例是從URL字串的自訂參數中提取值。 如果您使用UTM以外的參數，但想要將值剖析出接觸點欄位，這個功能會很實用。

**連結：** `https://www.adobe.com/blog/marketing-revenue-reporting-overview?promo=5OFF` 或 `https://www.adobe.com/blog/marketing-revenue-reporting-overview?promo=25OFF`.\
**目標：** 建立名為「折扣代碼」的自訂欄位，然後捨入值「5OFF」或「25OFF」，無論傳遞什麼值。

* 建立計算欄位並將其標示為「折扣代碼」
* 從搜尋Touchpoint.Session.LandingPage欄位開始，定義規則
* 請使用運算子「擷取」，因為我們需要從參數中提取值
* 要提取促銷活動的值，我們將該值定義為&quot;promo=(\w+)&quot;

   * **(** 會標示擷取的開始
   * **)** 會標示擷取的結尾
   * **\w** 告訴我們，我們提取的單詞包括0-9
   * **+** 會擷取參數的完整值，字元上沒有限制
   * 請注意，您使用的是正斜線，而非反斜線

**映射至**

映射到運算子將建立一個值表，這些值需要被轉換或分組到另一個值中。 通常會以索引鍵值的形式出現，其中程式碼代表好記名稱，且必須對應至該好記名稱。

**範例#1**

您已針對「夏季結束促銷活動」和「黑色星期五促銷活動」建立多個管道的促銷活動。 您想要建立名為「方案」的計算欄位，且除了其他可能的值外，還想將具有「夏季結束促銷活動」或「黑色星期五促銷活動」的任何接觸點對應至方案值，例如「促銷活動」。

![](assets/six.png)

**目標：** 建立多個計算欄位，其中第一個數字映射至地區，第二個數字映射至產品，第三個映射至計畫，第四個映射至角色，第五個映射至媒體平台。 接著，將數值對應至「好記的名稱」。

   * **)** 會標示擷取的結尾

      * 請注意，由於我們只擷取4，因此只有前幾位帶有右括弧

   * **{2}** 是我們提取的字元數



* 按一下 [!UICONTROL Save]. 您必須先儲存新欄位，才可供下一個規則使用！
* 接下來，我們會將頭位的所有可能值對應至好記名稱
* 建立計算欄位，並將其標籤為「Region_Name」
* 從搜尋擷取的欄位開始，定義規則。 在此案例中， Touchpoint.Region
* 使用運算子「[!UICONTROL maps to]&quot;，因為我們想為每個數字建立到其值的映射
* 您將看到一個表格，列出每個對應。 最後，它看起來會像這樣：
* 根據對應和上述URL，此登錄頁面之接觸點的「Region_Value」應為「EMEA」

**串連**

串連運算子會將多個欄位的值合併為單一欄位。 這對於建立自訂值很實用，該自訂值可跨不同欄位提取資料，以便建立

**範例#1**

Segment__c和Grade__c的Opportunity對象上有單獨的欄位，用戶希望將這些欄位合併到Touchpoint對象上的單個欄位，以用於報告目的。 串連欄位後，您就會看到Enterprise_A或Mid-Market_B之類的值。

## 接觸點欄位和區段 {#touchpoint-fields-and-segments}

現在，您的URL中的值已剖析完畢且存在於接觸點上，只要使用接觸點欄位（例如建立區段或定義接觸點刪除規則），您就會看到新欄位。

此產品版本提供使用接觸點欄位建立區段的功能。 無法先使用接觸點欄位建立區段。

![](assets/nine.png)

為了更輕鬆建立區段，您現在可以從已建立的接觸點欄位建立動態區段。 例如，如果您建立了剖析出地理區域的接觸點欄位，而不是為每個可能的區域建立區段，則您可以設定一個區段，我們將為每個例項建立區段，並顯示新值。 如果需要剖析屬性（例如郵遞區號）並將其用作區段，這個功能會非常實用！

您的設定看起來會像下方的螢幕截圖。 「區段名稱」會使用大括弧以動態方式提取接觸點欄位值，以搜尋您的欄位。

![](assets/ten.png)

規則會參照相同的接觸點欄位，並搜尋「不等於null」的值。

## 常見問題集 {#faq}

**我們可以建立的接觸點欄位數上限嗎？**

欄位上限為100個。

**我在選擇清單中看不到剛建立的新接觸點欄位。 它在哪裡？**

建立規則後，別忘了儲存規則。 如果您沒有看到新欄位，請檢查您是否已儲存。 您必須先儲存新欄位，才可供下一個規則使用。

>[!NOTE]
>
>由於複雜程度，使用「對應至」運算子的接觸點欄位無法用於其他接觸點欄位。

**我要使用什麼運算式從單一登陸頁面擷取多個參數？**

如同在擷取範例#4中，您需要建立多個欄位以擷取每個參數。 因此，如果您有5個不同的值，您將建立5個接觸點欄位來擷取每個值。

**為什麼我在 [!DNL Marketo Measure] 綱要？**

需要進行其他工作來公開 [!DNL Marketo Measure] Data Warehouse結構。 目前，欄位會透過設定和設定公開，以便您在建立區段或建立接觸點刪除規則時，使用接觸點欄位。

**如何驗證我的擷取運算式是否有效並提取正確的值？**

有線上工具([https://regex101.com/](https://regex101.com/){target="_blank"})，以執行並測試運算式。 如果運算式有效，則顯示為綠色；如果無效，則顯示為紅色。 此外，右上角的說明方塊很實用，可告訴您擷取的內容。
